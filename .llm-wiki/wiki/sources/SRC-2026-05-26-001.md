# 2026-05-25-skybox-ecosystem-gap-analysis.md

## Summary

A comprehensive gap analysis of the SkyBox/MatchBox project against the full Ortus ecosystem (BoxLang, ColdBox, WireBox, CFML). The analysis was performed by the `ortus-scholar` subagent using official GitBook MCP endpoints and direct codebase inspection.

The report identifies three major deficiency areas: (1) MatchBox cannot run ColdBox applications at all — it lacks the ColdBox dispatcher, WireBox DI, interceptor chain, request context scopes (RC/PRC), SES URL routing, and view rendering. (2) Legacy CFML code (`.cfc`/`.cfm` files) is completely unsupported — the MatchBox parser only handles `.bx`/`.bxs` syntax, the CFML transpiler is Java-only, and the `bx-compat-cfml` runtime shims are JVM-only. (3) The BIF registration and module system is compile-time only, with no dynamic runtime module loader, no interceptor/event system, no DI container, and no way to register BIFs from BoxLang code at runtime.

The analysis rates each gap by severity (Critical/High/Medium) and provides detailed "what would be needed" sections for each area, along with cross-cutting priority recommendations. The overall assessment: solid low-level foundation (VM, BIF registration, async callout bridge), but the compile-time-only architecture is the primary constraint blocking ecosystem compatibility.

## Key Takeaways

- **ColdBox on Cloudflare is not feasible without major framework shims** — 7 major components (Controller, WireBox, Interceptors, RC/PRC, SES Router, View Renderer, Module System) would need to be rebuilt from scratch
- **Legacy CFML support requires a Rust-based CFML parser** — the JVM transpiler cannot run on WASM; this is the hardest single dependency (multi-month effort)
- **The compile-time-only module/BIF system is the biggest architectural constraint** — no runtime discovery, no dynamic registration, no interceptor hooks
- **P1 priorities**: Add dynamic BIF registration at runtime and build a CFML parser; these unlock the most downstream value
- **Hybrid approach recommended**: Pre-compile CFML at build time, keep compile-time module resolution for BIFs
