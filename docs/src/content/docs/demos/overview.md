## Chatroom Web UI

The chatroom demo includes a Web UI served dynamically via `onHttpGet`. The HTML page is generated by the BoxLang listener (`ChatRoom.bx` → `renderChatPage()`), while CSS and JS are served as external static assets via Cloudflare's `[assets]` feature:

| Asset | Source | Served at |
|-------|--------|-----------|
| CSS | `assets/css/style.css` | `/css/style.css` |
| JS | `assets/js/chat.js` | `/js/chat.js` |

The `build-multi.sh` script copies `assets/` → `dist/assets/` during the build, and `wrangler.toml` has `[assets] directory = "dist/assets"` to serve them at the edge.

## Building Demos

All demos are built via the standard build pipeline:

```bash
# Single-file demos (echo, counter, moonphase, etc.)
bash examples/build.sh examples/<demo> <listener>.bx <ListenerClass>

# Multi-file demo with assets (chatroom)
bash examples/chatroom/build-multi.sh
```
